# 環境建置 WBS 與教育訓練計畫

## 1. 概述

本文件提供 NVIDIA Run.ai 平台建置的工作分解結構（WBS），以及對應的人員教育訓練計畫，確保專案可依序推進並培養內部維運能力。

---

## 2. 工作分解結構（WBS）

### 2.1 WBS 總覽

```
1.0 Run.ai × K8s 平台建置專案
 ├── 1.1 前置規劃
 ├── 1.2 基礎設施建置
 ├── 1.3 Kubernetes 叢集建置
 ├── 1.4 GPU 環境建置
 ├── 1.5 Run.ai 平台安裝
 ├── 1.6 整合服務建置
 ├── 1.7 測試與驗收
 └── 1.8 上線與移轉
```

#### 1.3 Kubernetes 叢集建置

> ⚠️ **Run.ai v2.25 新增前置需求**：平台預設採用 host-based routing，需在 K8s 叢集就緒前完成 LoadBalancer、FQDN / Wildcard DNS 及三組 TLS 憑證的配置。三組憑證依 v2.25 Cluster System Requirements 定義為：① cluster domain 憑證、② workspaces/training wildcard 憑證、③ inference wildcard 憑證。缺少任一項將導致對應功能無法正常存取。

| 工作項目 | 負責人 | 預估工時 | 交付物 |
|----------|--------|----------|--------|
| 1.3.1 kubeadm / RKE2 安裝（Control Plane HA × 3，目標 K8s 1.33–1.35） | K8s 工程師 | 2 天 | 叢集建置文件 |
| 1.3.2 Worker Node 加入叢集 | K8s 工程師 | 1 天 | 節點清單 |
| 1.3.3 CNI 安裝（Calico / Cilium） | K8s 工程師 | 1 天 | 網路連通測試 |
| 1.3.4 CSI Driver 安裝（NFS/Ceph） | K8s 工程師 | 1 天 | StorageClass 測試 |
| 1.3.5 LoadBalancer 安裝與設定（MetalLB / HAProxy / 硬體 LB） | 網路工程師 | 1.5 天 | LB IP Pool 分配與連通測試報告 |
| 1.3.6 Ingress Controller / Gateway API 安裝（含 host-based routing 設定） | K8s 工程師 | 1.5 天 | host-based routing 端對端測試 |
| 1.3.7 FQDN / Wildcard DNS 規劃與設定（*.runai.domain.local） | 網路工程師 | 1 天 | DNS 解析測試報告 |
| 1.3.8 TLS 憑證申請與安裝（① cluster domain 憑證、② workspaces/training wildcard 憑證、③ inference wildcard 憑證，共 3 組） | 資安工程師 | 1.5 天 | 三組憑證有效性確認單 |
| 1.3.9 cert-manager 安裝（TLS 憑證自動續期管理） | K8s 工程師 | 1 天 | 憑證自動更新測試 |
| 1.3.10 etcd 備份設定 | K8s 工程師 | 0.5 天 | 備份排程設定 |

## 3. 教育訓練計畫

### 3.1 訓練對象與課程規劃

#### A. 平台管理員（2–3 人）

| 課程模組 | 時數 | 內容摘要 | 形式 |
|----------|------|----------|------|
| K8s 核心概念 | 8h | Pod/Deployment/Service/RBAC/Namespace | 講師授課 |
| K8s 進階管理 | 8h | etcd 備份、節點管理、Helm、故障排查 | 講師授課 |
| Run.ai 管理員操作 | 8h | Department/Project 設定、配額管理、使用者管理 | 講師授課 + 實作 |
| GPU Operator 維護 | 4h | 驅動升級、DCGM 監控、MIG 設定 | 講師授課 |
| 監控與告警 | 4h | Prometheus/Grafana/Alertmanager | 講師授課 + 實作 |
| 升級 SOP 演練 | 4h | 依本文件 SOP 實際演練 | 動手實作 |
| **合計** | **36h** | | |

#### B. MLOps / 平台工程師（3–5 人）

| 課程模組 | 時數 | 內容摘要 | 形式 |
|----------|------|----------|------|
| Run.ai CLI 基礎 | 4h | runai submit / list / describe / logs | 實作 |
| 工作負載類型實作 | 6h | Training / Interactive / Distributed / Inference | 動手實作 |
| GPU Fraction 操作 | 2h | 分數申請、共享設定 | 動手實作 |
| Harbor 映像檔管理 | 2h | Push / Pull / 漏洞掃描 | 實作 |
| ArgoCD GitOps | 4h | Application 部署、同步策略 | 實作 |
| MLflow 整合 | 4h | 實驗追蹤、模型版本管理 | 實作 |
| **合計** | **22h** | | |

#### C. AI 研究人員 / 資料科學家（10–50 人）

| 課程模組 | 時數 | 內容摘要 | 形式 |
|----------|------|----------|------|
| Run.ai 使用者快速入門 | 2h | 登入、提交作業、查看狀態、查看 log | 線上影片 + 文件 |
| JupyterHub 互動開發 | 2h | Notebook 啟動、GPU 申請、資料掛載 | 實作 |
| 分散式訓練提交 | 2h | PyTorch DDP / Horovod 在 Run.ai 上執行 | 動手實作 |
| 儲存與資料集使用 | 1h | PVC 掛載、S3 存取 | 文件自學 |
| **合計** | **7h** | | |

---

### 3.2 訓練排程建議

```
Week 8–9（平台建置完成後）：
  Day 1–2：平台管理員深度訓練（K8s + Run.ai Admin）
  Day 3–4：MLOps 工程師操作訓練
  Day 5  ：AI 研究人員快速入門課程

Week 10（UAT 期間）：
  邊測試邊訓練，問題即時解決
```
